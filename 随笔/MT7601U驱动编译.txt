archlinux编译MT7601U驱动支持：
参考：http://www.takobear.tw/201702608526356260322804024687/rpi2arch-linuxwifi1

流程如下：
程式碼還有兩處要改
一處是因為Arch linux採用gcc的版本是4.9
有個地方gcc4.8是ok的，而4.9卻變成了Error，修改之
$ vim ./sta/sta_cfg.c
第5766行，在vim裡可用 " / " 來搜尋
ex: /__DATE__
snprintf(extra, size, "Driver version-%s, %s %s\n", STA_DRIVER_VERSION, __DATE__, __TIME__ );
改成
snprintf(extra, size, "Driver version-%s, (date and time removed for sanity)\n", STA_DRIVER_VERSION );
參考資料
另一處則是有地方的變數沒有宣告
$ vim ./os/linux/rt_linux.c
第1121、1122行，可用 " /current_fsuid() " 尋找
pOSFSInfo->fsuid = current_fsuid();
pOSFSInfo->fsgid = current_fsgid();
改成
pOSFSInfo->fsuid = (int )&current_fsuid();
pOSFSInfo->fsgid = (int )&current_fsgid();

实际根据make得到的提示，发现需要有三处，还缺少的一处为：
/common/cmm_info.c文件的47行，和1中的修改类似，将：
DBGPRINT(RT_DEBUG_TRACE, ("Driver version-%s %s %s\n", STA_DRIVER_VERSION, __DATE__, __TIME__));
改为
DBGPRINT(RT_DEBUG_TRACE, ("Driver version-%s, (date and time removed for sanity)\n", STA_DRIVER_VERSION ));


然后就可以通过编译了。
make
sudo make install
sudo modprobe mt7650u_sta 或者 mt7610u_sta 根据编译出来的ko文件确定 mt7601Uap
sudo depmod
ip addr

出问题，卸载驱动模块：
切换到驱动源代码路径下：
sudo make uninstall
make clean
检查当前模块是否加载
lsmod | grep mt7650u_sta
如果存在，删除驱动：
sudo modprobe -r mt7650u_sta
刷新：
sudo depmod


安装aricrack-ng：
sudo service networking restart
ifconfig ra0 up
